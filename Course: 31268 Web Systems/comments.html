Personal Reflection

I'm happy with the way my website turned out. I wanted a clean, modern design that effectively showcases everything abotu me. Using vibrant backgrounds and a simple layout creates a visually appealing experience without sacrificing easy navigation and readability. I think I achieved a good balance between aesthetics and functionality, making the website both engaging and informative.

Technical Aspects

Structure

Looking at the HTML code, you can see I used divs as the main building blocks for the layout. Each page (index.html, past.html, future.html) follows a similar structure with a header, main content area, and footer. For instance, the header in index.html is created with the header tag and contains the h2 tag for my name and the nav element with an unordered list (ul) for the navigation links.

CSS Selectors

I used classes like .main, .title, and .info to style elements that appear on multiple pages. You can see this in websystems.css. For example, the .info class styles the boxes with my personal information and uses background: rgba(0, 0, 0, 0.293); to give them a semi-transparent black background.

Responsiveness

The website adapts to different screen sizes thanks to the meta tag with name="viewport" in each HTML file and media queries in the CSS. This is evident in the way the navigation bar changes its layout on smaller screens. For example, the media query @media (max-width: 600px) in websystems.css adjusts the font size of the header and navigation links for better readability on smaller devices.

Effects

I used CSS3 features like box-shadow (visible on the image captions) and border-radius to create rounded corners on elements like the information boxes and image thumbnails. The slight fade-in animation on page load is achieved using animation: fadeIn 2s; in the CSS, adding a subtle dynamic touch.

Aesthetics

Minimalism

The overall aesthetic is minimalist, with a focus on clean lines and ample white space. This is evident in the layout of the text and images, particularly on pages like "My Past" where the information is presented in clear sections with space between them. The color palette of black, white, and grey creates a neutral base that allows the vibrant background images to stand out.

Typography

The choice of fonts, Roboto for headings and Open Sans for body text, ensures readability and creates a clear visual hierarchy. This is noticeable in sections like "Who am I?" where the heading is distinct from the paragraph text.

Visual Hierarchy

Headings like "HSC" and "Work Experience" on the "My Past" page guide the user's attention and provide a clear structure to the content. The use of paragraphs and line breaks further enhances readability.

Imagery

The personal photos, like the ones of my cat Daisy and my gaming setup, add personality and visual interest. The circular cropping of images in the "images-second" sections creates a cohesive look.

Accessibility

Alt Text

All images have alt text. For example, the images of Daisy have descriptions like "Daisy watching football" and "Daisy looking outside," which would be read by screen readers, allowing visually impaired users to understand the content.

Text Size and Contrast

The font size is set at 16px, ensuring comfortable reading. The black text against the light backgrounds and white text on dark backgrounds provides good contrast. For example, the white text on the information boxes is easily readable against the semi-transparent black background.

Semantic HTML

Using semantic elements like header, nav, and footer helps screen readers understand the structure of the page. This is evident in the consistent use of these elements across all pages.

Responsiveness

The responsive design ensures the website is accessible on various devices. This benefits users with different needs and preferences, allowing them to comfortably access the content on their chosen device.

Areas for Improvement

ARIA attributes

While alt text is used for images, adding ARIA attributes like aria-label could further improve accessibility for users with screen readers. For example, the navigation links and icons could benefit from more descriptive labels.

Color Contrast

While the color contrast is generally good, there are areas where it could be further enhanced. For instance, the white text overlaid on the brighter sections of the background images might be difficult to read for some users. Adjusting the text color or adding a subtle background behind the text could improve readability.

Keyboard Navigation

Implementing keyboard navigation would allow users who rely on keyboards to navigate the site effectively. This includes ensuring all interactive elements like links and buttons can be accessed and activated using the keyboard.
